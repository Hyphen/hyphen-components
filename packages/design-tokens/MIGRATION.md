# Migration: Style Dictionary v3 → v5

This repo was upgraded from Style Dictionary `3.9.2` to `5.5.0` (v4 and v5 are
both major releases — see the official
[v4](https://styledictionary.com/version-upgrades/v4/) and
[v5](https://styledictionary.com/version-upgrades/v5/) upgrade guides).

The token names, file names, and file locations of everything published in the
`build/` directory are **unchanged**. `build/utilities/utilities.css`, the
React icon/logo components, and `build/types/index.d.ts` are identical to the
v3 output (apart from the generation timestamp some of their headers embed —
see section 5 below).

## Impact on consumers (hyphen-app, public-website)

**No action is required** for typical usage (importing the CSS/SCSS variable
files, utility classes, icon components, or referencing tokens by name).
A handful of token *values* in the generated output changed. All of them are
rendering-equivalent or bug fixes:

### 1. `transparent` / alpha colors are no longer mangled in JS and JSON output (bug fix)

In v3, the `color/hex` transform used for `build/js/variables.js` and
`build/json/variables.json` silently dropped the alpha channel:

| Token | v3 value (wrong) | v5 value |
| --- | --- | --- |
| `color.background.transparent`, `color.background.button-tertiary`, `color.background.badge-outline`, `color.background.scrollbar-track`, `color.border.transparent`, `color.border.button-tertiary{,-hover,-active}` | `#000000` | `transparent` |
| `color.background.toggle-group-item-hover`, `color.background.toggle-group-item-active` | `#000000` | `rgba(0, 0, 0, 0.1)` |

If any code reads these tokens from the JS/JSON exports and expects a
6-digit hex string, it now receives the real (previously lost) value. The CSS
and SCSS outputs always had the correct values, so anything styled via CSS
custom properties is unaffected.

### 2. Hex letter case is now preserved from the token source

Values such as `#ffffff` are now emitted as `#FFFFFF` (the casing used in
`tokens/`), in all output targets. CSS treats hex colors case-insensitively;
this only matters if code compares token values as strings (e.g.
`if (color === '#ffffff')`) or snapshots them in tests.

### 3. `transparent` is emitted as-is in CSS/SCSS

`rgba(0, 0, 0, 0)` → `transparent` for the tokens listed above. Computes
identically in browsers.

### 4. Metadata changes in JS/JSON exports

Each token object in `build/js/variables.js` and `build/json/variables.json`
now carries an additional `key` property (e.g. `"{color.base.white}"`), added
by Style Dictionary v5. Existing properties (`value`, `darkValue`, `name`,
`attributes`, `path`, `original`, …) are unchanged.

### 5. Style Dictionary file headers no longer contain a timestamp

Files generated by Style Dictionary's built-in formats (the CSS/SCSS/JS/JSON
variables files) now start with `Do not edit directly, this file was
auto-generated.` instead of a `Generated on <date>` line. Update any snapshot
tests that captured the old header.

Note: `build/utilities/utilities.css` and the React icon/logo `index.js`
files use this repo's own `createFileHeader` util and still include a
`Generated on <date>` timestamp, exactly as they did before the upgrade.

## Changes in this repo (for contributors)

- **Node.js >= 22 is required** (Style Dictionary v5 requirement). `.nvmrc`,
  `engines`, and CI were updated.
- **`build.js` → `build.mjs`**: Style Dictionary v4+ is ESM-only, so the build
  script is now an ES module. `new StyleDictionary(config)` replaces
  `StyleDictionary.extend(config)`, and `buildAllPlatforms()` is awaited.
- **Token references no longer use the `.value` suffix**:
  `{color.base.grey.700.value}` → `{color.base.grey.700}`. v5 resolves
  references against a token map keyed by token path, so `.value`-suffixed
  references no longer resolve. This applies to `darkValue` references too.
- **Hook API renames** (old names were removed in v5):
  - `registerTransform`: `matcher` → `filter`, `transformer` → `transform`
  - `registerFormat`: `formatter` → `format`; the function receives
    `({ dictionary, options, file, platform })` and token arrays are exposed
    as `dictionary.allTokens` (`allProperties` was removed)
  - config-level custom formats live under `hooks.formats` instead of `format`
  - built-in formats are looked up via `StyleDictionary.hooks.formats[name]`
    instead of `StyleDictionary.format[name]`
- **Built-in transform renames**: `name/cti/kebab` → `name/kebab`,
  `name/cti/pascal` → `name/pascal`, `content/icon` → `html/icon`.
- **`config.json` was removed**: it was an unused leftover (the build always
  ran through `build.js`) and still used v3 syntax.
- Platforms that only run actions (e.g. the `assets` platform with
  `copy_assets`) must declare `files: []` in v5.
